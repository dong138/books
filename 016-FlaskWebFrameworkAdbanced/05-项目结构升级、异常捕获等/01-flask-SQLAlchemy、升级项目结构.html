flask框架进阶

flask-SQLAlchemy、升级项目结构

1. 目的

  1. 之前每次使用SQLAlchemy都需要使用sessionmaker创建session，然后才可以使用，这样麻烦，使用flask-SQLAlchemy则可以更简单的使用SQLAlchemy

  2. 如果将所有的文件夹以及main.py一起暴露，很可能会看上去比较乱，所以将之前的项目结构再次进行包裹升级

2.2.1 创建flask-sqlalchemy扩展的SQLAlchemy对象

此时会有很多的异常信息，是因为：此时的SQLAlchemy需要flask应用对象中的配置，例如链接哪个数据库，用户名以及密码是多少等。想要解决这个问题，就需要对flask应用对象进行配置

2.2.2 对flask对象进行配置

将如下代码，添加到main.py中

# 数据库相关配置
# 设置数据库的链接地址
app.config["SQLALCHEMY_DATABASE_URI"] = 'mysql+pymysql://root:python@localhost:3306/flask_1?charset=utf8'
# 关闭追踪数据库的修改
app.config["SQLALCHEMY_TRACK_MODIFICATIONS"] = False

2.2.4 升级项目结构

修改app/__init__.py

from flask import Flask
from flask_sqlalchemy import SQLAlchemy

# 创建SQLAlchemy对象
db = SQLAlchemy()


def create_app():
    # 创建一个flask应用对象
    app = Flask(__name__)

    # 如果用到了session，那么就需要设置秘钥
    app.config["SECRET_KEY"] = "1"
    # 数据库相关配置
    # 设置数据库的链接地址
    app.config["SQLALCHEMY_DATABASE_URI"] = 'mysql+pymysql://root:python@localhost:3306/flask_1?charset=utf8'
    # 关闭追踪数据库的修改
    app.config["SQLALCHEMY_TRACK_MODIFICATIONS"] = False

    # 将蓝图注册到app上
    # 导入创建的蓝图
    from app.views.admin import admin_blu
    from app.views.index import index_blu
    app.register_blueprint(admin_blu, url_prefix="/admin")
    app.register_blueprint(index_blu)

    # 创建SQLAlchemy对象
    # db = SQLAlchemy(app)  # 需要将创建的flask对象当做参数进行传递，它会自动与flask对象进行关联
    db.init_app(app)  # 这种方式是先创建db，然后再将flask对象与之关联

    return app

修改app/models/models.py

from app import db

class User(db.Model):
    # 对应MySQL中数据表的名字
    __tablename__ = 'user'

    # 创建字段
    id = db.Column(db.Integer, primary_key=True)
    user_id = db.Column(db.String(50), nullable=False)
    user_name = db.Column(db.String(50), nullable=False)
    email = db.Column(db.String(100), nullable=False)
    password = db.Column(db.String(100), nullable=False)
    head_img = db.Column(db.String(200))
    short_description = db.Column(db.String(300))
