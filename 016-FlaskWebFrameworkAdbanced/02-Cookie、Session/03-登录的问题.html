3. 思考

既然点击登录时，已经提交了用户名、密码，那么就应该在视图函数中提取出来，然后与数据库中的用户名和密码进行对比，如果用户名和密码一致的话，那么就表示登录成功，那又应该怎样获取url中用户名和密码呢？

4. 提取url参数

4.1 修改views/index/views.py

from flask import render_template
from flask import request
from sqlalchemy.orm import sessionmaker

from models.models import User, engine
from . import index_blu


@index_blu.route("/profile_v8/<user_id>")
def profile_v8(user_id):
    # 1. 查询数据库
    # 创建session对象
    DBSession = sessionmaker(bind=engine)  # 创建与数据库的会话，返回的是一个类
    session = DBSession()  # 生成链接数据库的实例

    # 获取返回数据的第一行
    obj = session.query(User).filter(User.user_id == user_id).one()
    # 关闭session
    session.close()

    # 2. 模板渲染
    return render_template("index/profile.html", user_name=obj.user_name, head_img=obj.head_img, short_description=obj.short_description)


@index_blu.route("/login")
def login():
    ret = request.args
    print(ret)
    return render_template("index/login.html")

4.2 说明

只要通过request.args就可以得到flask已经处理好之后的url参数

requests.args表示的是一个字典，可以通过迭代的方式提取所有，或者通过get方法提取某个指定的参数

修改login方法如下:

@index_blu.route("/login")
def login():
    ret = request.args
    if ret:
        user_name = ret.get("Username")
        passwd = ret.get("Password")
        return "用户名:%s ，密码:%s" % (user_name, passwd)
    else:
        return render_template("index/login.html")

说明

第3行：提取所有的url中参数

第4行：判断是否提取到数据，如果有表示进行了真正的登录过程，如果没有表示访问了登录页面

第5、6行：如果url有参数，那么就提取Username、Password

第7行：将url中提取的参数返回直接让浏览器显示，以便测试（此时可以先不用模板，等测试成功后从网上找一个漂亮的登录结果的模板进行替换即可）

第8、9行：如果url没有参数，那么就表示用户访问login是为了看到登录的页面，所以将这个页面数据返回即可

from flask import render_template
from flask import request
from sqlalchemy.orm import sessionmaker
from sqlalchemy.sql import and_

from models.models import User, engine
from . import index_blu


@index_blu.route("/profile_v8/<user_id>")
def profile_v8(user_id):
    # 1. 查询数据库
    # 创建session对象
    DBSession = sessionmaker(bind=engine)  # 创建与数据库的会话，返回的是一个类
    session = DBSession()  # 生成链接数据库的实例

    # 获取返回数据的第一行
    obj = session.query(User).filter(User.user_id == user_id).one()
    # 关闭session
    session.close()

    # 2. 模板渲染
    return render_template("index/profile.html", user_name=obj.user_name, head_img=obj.head_img, short_description=obj.short_description)


@index_blu.route("/login")
def login():
    ret = request.args
    if ret:
        user_name = ret.get("Username")
        passwd = ret.get("Password")
        # 创建session对象
        DBSession = sessionmaker(bind=engine)  # 创建与数据库的会话，返回的是一个类
        session = DBSession()  # 生成链接数据库的实例
        # 获取返回数据的第一行
        try:
            obj = session.query(User).filter(and_(User.user_id == user_name, User.passwd == passwd)).one()
        except:
            msg = "登录失败...<br>"
        else:
            msg = "用户名、密码匹配，登录成功...<br>"
        # 关闭session
        session.close()
        return msg + "用户名:%s ，密码:%s" % (user_name, passwd)
    else:
        return render_template("index/login.html")

6.4 发现问题

如果电脑上还有其他版本的浏览器请打开，然后访问个人简历页面，请看看效果

当然还有其他的实验办法，你在火狐浏览器中故意用一个错误的账户登录，然后再重新回到谷歌浏览器中看到之前那个可以访问的页面又不能正确的访问了

也就是说：

如果采用定义一个全局变量的方式，不可取

原因：

只要flask程序运行起来，无论用多少个浏览器访问web服务器，web服务器都是同一个web程序程序，如果一个用户修改了这个全局变量，那么再用这个flask为其他的未登录的用户提供页面的时候，都当做登录之后的来处理了，

每一个用户都应该用自己独有的标记才可以，而不是所有用户共用这个同一个全局变量
