为了能够让浏览器获取到图片验证码，需要一个视图函数，如下

@index_blu.route("/captcha")
def captcha():
    from utils.captcha.captcha import captcha

    # 1. 获取到当前的图片编号id
    captcah_id = request.args.get('id')

    print(type(captcah_id), captcah_id)

    # 2. 生成验证码
    # hash值  验证码值  图片内容
    name, text, image = captcha.generate_captcha()

    # print("注册时的验证码为：", name, text, image)  # hash值  验证码值  图片内容

    # 3. 将生成的图片验证码值作为value，存储到session中
    session["captcha"] = text

    # 返回响应内容
    resp = make_response(image)
    # 设置内容类型
    resp.headers['Content-Type'] = 'image/jpg'
    return resp

2.5 完善登录时的验证

既然登录页面已经有了图片验证码，那么就应该在登录判断的逻辑中对图片验证码进行判断

核心过程：

  1. 浏览器获取图片验证码时，浏览器有图片、以及cookie的名字为session（例如其值是123123123123123123123）；而验证码的值(例如hjyw)却在服务器中存储
  2. 浏览器提交session(123123123123123123123)以及用户输入的验证码给服务器(hjyw)
  3. 服务器通过sessioin(123123123123123123123)提取出验证码值hjyw，与用户提交的验证码值hjyw进行对比，如果相同则认为验证码正确，否则不正确
