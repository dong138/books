切片

1. 切片（slice）概念

1.1 引入

在讲解切片（slice）之前，大家思考一下数组有什么问题？

  • 数组定义完，长度是固定的。如果再向数组num中追加数字，这时会出错。
  • 使用数组作为函数参数进行传递时，如果实参为5个元素的整型数组，那么形参也必须5个元素的整型数组，否则出错

针对以上两个问题，可以使用切片来进行解决

1.2 什么是切片？

切片是一种存储多个数据的方式

可以将切片理解成动态数组，但是，它不是数组

切片与数组相比：切片的长度是不固定的，可以追加元素，在追加时可能使切片的容量增大

1.3 切片与数组区别

通过定义，来比较一下切片与数组的区别

数组的定义

a := [5]int{11, 22, 33, 44, 55}
  • 数组中[ ]是一个固定的数字，表示长度
  • 定义完后，长度是固定，最多存储5个数字

切片的定义

s := []int{}  // 定义空切片

通过上述对比发现切片与数组很相似，但是注意：

  • 切片中的[]是空的
  • 数组中的[]是一个固定的值

1.4 demo

现在通过程序演示，动态向切片中追加数据

package main

import "fmt"

func main()  {
    s := []int{11, 22, 33}
    fmt.Println("切片s的长度是:", len(s), "内容是:", s)

    s = append(s, 44, 55, 66)
    fmt.Println("切片s的长度是:", len(s), "内容是:", s)
}

运行效果

说明：

  • append( )函数，第一个参数表示向哪个切片追加数据，后面表示具体追加的数据。

2. 定义切片的其它方式

方式2

var s1 []int  // 声明切片和声明数组一样，只是少了长度，此为空(nil)切片

方式3

通过make( )函数实现

// 借助make函数, 格式 make(切片类型, 长度, 容量)
s := make([]int, 5, 10)

在使用make( )函数定义切片时，一定要注意，切片长度要小于容量，

例如：

s := make([]int, 10, 5)  // 是错误的

make( )函数中的容量参数是可以省略掉的，此时这时长度与容量是相等的

例如：

s := make([]int, 10)  // 省略了容量

3. 长度与容量

3.1 什么是切片的长度与容量？

长度

是已经初始化的空间

容量

是已经开辟的空间 （包括已经初始化的空间和空闲的空间）

理解切片的长度与容量：

  • 该切片的长度是5（存有数据，注意如果没有赋值，默认值都是0）
  • 容量是10，只不过有5个空闲区域

demo

package main

import "fmt"

func main()  {
    // 创建切片
    s := make([]int, 5, 10)

    // 查看默认的数据
    fmt.Println(s)

    // 长度
    fmt.Println("s的长度是:", len(s))

    // 容量
    fmt.Println("s的容量是:", cap(s))
}

运行效果

注意：

  • len()测量长度
  • cap()测量容量

package main

import "fmt"

func main()  {
    // 定义切片，且完成初始化
    s := []int{11, 22, 33, 44, 55, 66}

    // 从切片s中截取数据
    s2 := s[0:3:5]

    // 打印s2
    fmt.Println("s2 =", s2)
    fmt.Printf("len(s2) = %d, cap(s2) = %d", len(s2), cap(s2))
}

运行效果

s[0:3:5]是什么意思呢？

我们可以使用s[low:high:max]来表示

  • 第一个数（low）表示下标的起点（从该位置开始截取），如果low取值为0表示从第一个元素开始截取
  • 第二个数（high）表示取到哪结束，也就是下标的终点（不包含该位置）
  • 第三个数（max）用来计算容量，通过公式cap=max-low计算 即5-0，也就是第三个数据减去第一个数。该案例中容量为5

s[0:3]与s[0:3:5]的不同

package main

import "fmt"

func main()  {
    // 定义切片，且完成初始化
    s := []int{11, 22, 33, 44, 55, 66}

    // 从切片s中截取数据
    s2 := s[0:3:5]

    // 打印s2
    fmt.Println("s2 =", s2)
    fmt.Printf("len(s2) = %d, cap(s2) = %d\n", len(s2), cap(s2))

    // 从切片s中截取数据
    s3 := s[0:3]

    // 打印s3
    fmt.Println("s3 =", s3)
    fmt.Printf("len(s3) = %d, cap(s3) = %d\n", len(s3), cap(s3))
}

运行效果

注意：

  • s[0:3:5]容量是5-0即5个

  • s[0:3]容量是6-0即6个，即如果没有指定容量，那么容量的大小为从起始下标一直到最后一个元素的个数

package main

运行效果

说明：

  • 追加完成一个数据后，长度变为9
  • 由于大于创建切片s时的容量，所以切片s扩容，变为16.

那么切片的容量是否是以2倍容量来进行扩容的呢？

我们可以来验证一下：

package main

import "fmt"

func main()  {

    var oldCap int

    s := make([]int, 0, 1)
    fmt.Println("s的容量是", cap(s))
    fmt.Println("------------------")

    oldCap = cap(s)

    for i:=0; i<2000; i++{
        s = append(s, i)

        if cap(s) != oldCap{
            oldCap = cap(s)
            fmt.Println("s的容量是", cap(s))
            fmt.Println("------------------")
        }

    }
}

运行效果

结论：

  • 当容量小于1024时是按照2倍容量扩容，当大于等于1024是不是按照2倍容量扩容。

7. copy函数使用

针对切片操作常用的方法除了append( )函数以外，还有copy函数

基本语法

copy(切片1，切片2)

作用：

  • 将第二个切片里面的元素，拷贝到第一个切片中

demo

package main

import "fmt"

func main()  {
    s1 := []int{11, 22}
    s2 := []int{33, 44, 55, 66, 77}

    // 将s1 copy 到s2
    copy(s2, s1)
    fmt.Println("s1 =", s1, "\ns2 =", s2)

    fmt.Println("-----------------------")

    s1 = []int{11, 22}
    s2 = []int{33, 44, 55, 66, 77}

    // 将s2 copy 到s1
    copy(s1, s2)
    fmt.Println("s1 =", s1, "\ns2 =", s2)

}

运行效果

结论：

  • 在进行拷贝时,拷贝的长度为两个slice中长度较小的长度值。

8. 切片作为函数参数

切片也可以作为函数参数，那么与数组作为函数参数有什么区别呢？

接下来通过一个案例，演示一下切片作为函数参数。

package main

import "fmt"

func Test(temp []int) {
    fmt.Println("在Test函数中，temp =", temp)
    temp[0] = 100
    fmt.Println("在Test函数中，temp =", temp)
}

func main()  {
    s := []int{11, 22, 33, 44, 55, 66, 77}

    fmt.Println("调用Test函数之前s =", s)

    Test(s)

    fmt.Println("调用Test函数之后s =", s)
}

运行效果

说明：

  • 在Test( )函数中对形参切片修改，影响到了main( )函数中的切片s

回顾数组当做参数时，是否也是会在被调用函数中修改，原函数中发送变化呢？

package main

import "fmt"

func Test(temp [7]int) {  // 形参发生了变化
    fmt.Println("在Test函数中，temp =", temp)
    temp[0] = 100
    fmt.Println("在Test函数中，temp =", temp)
}

func main()  {
    s := [...]int{11, 22, 33, 44, 55, 66, 77}  // 这里发生了变量

    fmt.Println("调用Test函数之前s =", s)

    Test(s)

    fmt.Println("调用Test函数之后s =", s)
}

说明

  • 如果数组当做参数，在被调用函数对数组进行修改，并不会影响到原函数中的数组

结论

  • 数组作为参数进行传递是值传递
  • 而切片作为参数进行传递是引用传递

9. 值传递？引用传递？

值传递

方法调用时，实参数把它的值传递给对应的形式参数，方法执行中形式参数值的改变不影响实际参数的值

直白点说：传递过去的是数据的备份

引用传递（也称地址传递）

函数调用时，实际参数的引用(数据在内存的地址，而不是参数的值)被传递给函数中相对应的形式参数，在函数执行中，对形式参数的操作实际上就是对实际参数的操作，方法执行中形式参数值的改变将会影响实际参数的值。

直白点说：传递过去的是数据的地址

案例

需求：

计算出一组整型数据之和。

参考代码：

package main

import "fmt"

func GetNums(temp []int) {
    for i:=0; i<len(temp); i++{
        fmt.Printf("请输入第%d个数:", i+1)
        fmt.Scanf("%d", &temp[i])
    }
}

func AddNums(temp []int) (ret int) {
    for _, num := range temp {
        ret += num
    }
    return
}

func main()  {
    var numsCount, sumResult int
    // 打印提示
    fmt.Println("请输入要计算的数据的个数：")
    // 获取个数
    fmt.Scanf("%d", &numsCount)
    // 定义切片
    s := make([]int, numsCount)
    // 获取指定个数的数据
    GetNums(s)
    // 计算求和
    sumResult = AddNums(s)
    // 打印结果
    fmt.Println("所有数据的和为", sumResult)
}

运行效果：
