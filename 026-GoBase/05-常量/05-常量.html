常量

1. 常量定义使用

在讲解常量之前，我们要先简单说一下我们前面讲解的变量。我们都知道，变量是用来存储数据，并且变量里面存储的值是可以在程序运行时随时发生变化的，例如如下案例，结果输出是多少？

package main

import "fmt"

func main()  {
    var num int
    num = 10
    num = 20
    fmt.Printf("num的值是:%d", num)
}

这个程序我们前面已经做过，最终的输出结果是20，因为重新给number变量赋值。

但是，常量里面的值确定好后，后面是不允许修改的。

看下面关于常量的定义使用：

package main

import "fmt"

func main()  {
    // 变量, 程序运行期间，可以修改，使用var声明
    // 常量, 程序运行期间，不可以修改，使用const声明
    const a int = 10
    a = 20
    fmt.Println("a=", a)
}

将第9行代码屏蔽之后

package main

import "fmt"

func main()  {
    // 变量, 程序运行期间，可以修改，使用var声明
    // 常量, 程序运行期间，不可以修改，使用const声明
    const a int = 10
    // a = 20  // 会出错，因为a是const修饰的，不能修改
    fmt.Println("a=", a)
}

常量的命名规范与变量命名规范一致。

问题：什么场景下会用到常量呢？

在程序开发中，我们用常量存储一直不会发生变化的数据，例如：π，身份证号码等。像这类的数据，在整个程序中运行中都是不允许发生改变的。

3. iota枚举

常量声明可以使用iota常量生成器初始化，它用于生成一组以相似规则初始化的常量，但是不用每行都写一遍初始化表达式。

注意：在一个const声明语句中，在第一个声明的常量所在的行，iota将会被置为0，然后在每一个有常量声明的行加1

具体使用方式如下：

package main

import "fmt"

func main()  {
    // iota常量自动生成器，每一行都会自动加1
    const (
        a = iota  // 0
        b = iota  // 1
        c = iota  // 2
    )

    fmt.Printf("a=%d, b=%d, c=%d\n", a, b, c)  // a=0, b=1, c=2

    // 当iota遇到const声明的变量时，它的值会重置为0
    const d = iota  // 0

    fmt.Printf("d=%d\n", d)  // d=0

    // 可以只写一个iota，后面的const常量会自动加1
    const (
        e = iota  // 0
        f  // 1
        g  // 2
    )

    fmt.Printf("e=%d, f=%d, g=%d\n", e, f, g)  // e=0, f=1, g=2

    // 如果在同一行中，那么iota的值相同
    const (
        A = iota  // 0
        B, C = iota, iota  // 1 1
        D = iota  // 2
    )

    fmt.Printf("A=%d, B=%d, C=%d, D=%d\n", A, B, C, D)  // A=0, B=1, C=1, D=2
}
